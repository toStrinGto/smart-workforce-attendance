# 智工考勤主流程测试文档

## 1. 测试目标

从登录页开始，覆盖智工考勤当前前端的四类角色主流程：

- 工人：登录、打卡、考勤日历、统计、工作台、日报、报销。
- 班组长：登录、批量记工、异常处理、工作台、日报、报销。
- 老板：登录、企业看板、项目考勤、人员考勤、报销审批、合同/结算/成本入口。
- 管理员：登录、后台工作台、项目管理、员工管理、考勤管理、系统设置。

本测试以本地 Mock 模式为主，验证页面路由、关键控件、主要交互、基础数据加载和退出登录流程。

## 2. 测试环境

- 本地地址：`http://localhost:3000`
- 登录页：`http://localhost:3000/login`
- 推荐启动方式：

```powershell
$env:VITE_MOCK_ENABLED='true'; npm run dev
```

说明：当前 `.env` 中 `VITE_MOCK_ENABLED=false`，若本地没有后端服务，必须用上面的环境变量覆盖为 Mock 模式，否则登录和列表数据会依赖真实后端。

## 4. 通用登录与鉴权测试

### TC-AUTH-001 未登录拦截

1. 清空浏览器本地存储。
2. 访问 `/`。
3. 预期跳转到 `/login`，页面显示“账号登录”“请输入手机号”“请输入密码”。
4. 访问 `/admin`。
5. 预期同样跳转到 `/login`。

### TC-AUTH-002 登录表单校验

1. 在登录页输入非法手机号，如 `123`，密码输入 `123456`。
2. 点击“登录”。
3. 预期显示“请输入正确的11位手机号”。
4. 输入合法手机号，密码输入少于 6 位。
5. 预期显示“密码至少6位”。

### TC-AUTH-003 错误账号或密码

1. 输入任一测试手机号，密码输入非 `123456`。
2. 点击“登录”。
3. 预期显示“手机号或密码错误”。

### TC-AUTH-004 角色登录跳转

分别使用四个测试账号登录：

- 工人、班组长、老板登录后应进入 `/`。
- 管理员登录后应进入 `/admin`。
- 登录后刷新页面应保持登录态。
- 点击“退出”后应回到 `/login`。

## 5. 工人端主流程

### TC-WORKER-001 打卡首页

1. 使用工人账号登录。
2. 预期顶部显示用户“张三”，页面主体显示“智工考勤”“当前项目: 绿地中心二期”“上班打卡”“今日打卡记录”。
3. 点击“上班打卡”。
4. 预期按钮进入拍照/定位加载状态，约 1.5 秒后显示“打卡成功！辛苦了”，今日记录新增“上班打卡 / 正常”。
5. 再次点击“下班打卡”。
6. 预期今日记录新增“下班打卡 / 正常”。

### TC-WORKER-002 考勤与统计

1. 点击底部“考勤”。
2. 预期进入 `/attendance`，显示“我的考勤”和月份切换按钮。
3. 点击上/下月切换。
4. 预期月份文案变化，无页面错误。
5. 点击底部“统计”。
6. 预期进入 `/stats`，显示“考勤统计”和统计卡片。

### TC-WORKER-003 工作台、日报、报销

1. 点击底部“工作台”。
2. 预期进入 `/workbench`，显示“常用应用”。
3. 点击“日报”。
4. 预期进入 `/daily-report`，显示“施工日报”“写日报”“历史记录”“模板管理”。
5. 返回工作台，点击“报销”。
6. 预期进入 `/reimbursement`，显示“费用报销”“申请报销”“报销记录”。
7. 在报销申请中输入金额和事由，点击“提交报销”。
8. 预期显示提交成功反馈，或页面保持可操作且无错误。

## 6. 班组长端主流程

### TC-FOREMAN-001 批量记工

1. 使用班组长账号登录。
2. 预期进入批量记工页，显示“批量记工”、当前项目卡片、工人列表和“全选”。
3. 点击“全选”。
4. 预期选中全部工人，计数变为全部人数。
5. 调整“白班”“加班”加减按钮。
6. 点击“提交记工”。
7. 预期显示提交成功反馈，已提交工人出现记录状态，选中项随后清空。

### TC-FOREMAN-002 切换项目

1. 在批量记工页点击项目卡片或“切换”。
2. 预期打开项目选择弹窗。
3. 选择另一个项目。
4. 预期当前项目名称更新，工人列表重新加载或保持可操作。

### TC-FOREMAN-003 异常处理

1. 点击底部“异常处理”。
2. 预期进入 `/exceptions`，显示“异常处理”和“待处理 / 已处理 / 已驳回”标签。
3. 在待处理列表点击“处理”。
4. 预期打开“处理异常”弹窗。
5. 选择“通过”，填写备注，点击确认。
6. 预期该异常状态更新或从待处理列表移除。
7. 再打开另一个待处理项，选择“驳回”，填写驳回原因后提交。
8. 预期状态更新到已驳回。

### TC-FOREMAN-004 工作台入口

1. 点击底部“工作台”。
2. 预期显示“日报”“报销”“排班”“设置”等入口。
3. 验证“日报”和“报销”能分别进入对应页面。

## 7. 老板端主流程

### TC-BOSS-001 企业看板

1. 使用老板账号登录。
2. 预期首页显示“企业看板”“项目情况”“各项目报销情况”以及收入合同、收入结算、收款金额、开票金额等数据卡片。
3. 点击“收入合同”卡片。
4. 预期进入 `/boss/contracts?tab=income` 或合同管理页。
5. 返回首页，点击“项目成本”。
6. 预期进入“项目成本统计”页面。

### TC-BOSS-002 项目考勤

1. 点击底部“考勤”。
2. 预期进入 `/attendance`，显示“项目考勤概览”和项目搜索框。
3. 点击“今日总出勤 (人)”统计卡片。
4. 预期进入 `/employee-attendance`，显示人员列表、全部/已打卡/未打卡筛选。
5. 点击某个员工。
6. 预期进入员工考勤明细页，显示员工摘要和考勤记录。
7. 返回项目考勤，点击某个项目。
8. 预期进入项目考勤详情页，显示项目名称、出勤/缺勤/加班统计和人员列表。

### TC-BOSS-003 报销审批

1. 点击底部“报销”。
2. 预期进入费用报销页，老板角色可见“申请报销 / 审批报销 / 报销记录”。
3. 点击“审批报销”。
4. 预期显示待审批报销列表。
5. 点击“通过”或“驳回”。
6. 预期列表项减少或状态改变，无页面错误。

### TC-BOSS-004 工作台管理入口

1. 点击底部“工作台”。
2. 预期显示“日报”“收入合同”“收入结算”“报销概览”“项目成本”等入口。
3. 分别点击收入合同、收入结算、报销概览、项目成本。
4. 预期都能进入对应页面并显示核心标题。

## 8. 管理员后台主流程

### TC-ADMIN-001 后台登录与导航

1. 使用管理员账号登录。
2. 预期进入 `/admin`，显示“智工考勤后台”“管理后台”“工作台”。
3. 左侧菜单应包含“工作台 / 项目管理 / 员工管理 / 考勤管理 / 系统设置”。

### TC-ADMIN-002 工作台

1. 在 `/admin` 查看统计卡片。
2. 预期显示“在建项目”“今日在岗”“考勤异常”“本月工时”。
3. 点击“在建项目”或“查看全部”。
4. 预期跳转到“项目管理”。

### TC-ADMIN-003 项目管理

1. 点击左侧“项目管理”。
2. 预期显示“项目管理”“搜索项目名称或负责人”“项目列表”。
3. 点击“新建项目”。
4. 预期打开“新建项目”弹窗。
5. 填写项目名称、项目经理、开工日期、预算等字段，提交。
6. 预期列表新增项目或出现成功反馈，无页面错误。
7. 点击某项目“编辑”“删除”，预期对应弹窗或确认框可用。

### TC-ADMIN-004 员工管理

1. 点击左侧“员工管理”。
2. 预期显示“员工管理”“搜索姓名或角色”“员工列表”。
3. 点击某员工“编辑”。
4. 修改日薪后点击“保存”。
5. 预期保存成功或列表值更新，无页面错误。

### TC-ADMIN-005 考勤管理

1. 点击左侧“考勤管理”。
2. 预期显示“员工考勤 / 项目考勤 / 异常管理”标签。
3. 在员工考勤中点击“详情”。
4. 预期打开员工考勤详情弹窗。
5. 在项目考勤中点击项目行。
6. 预期打开项目考勤详情弹窗。
7. 在异常管理中处理异常。
8. 预期通过/驳回流程可执行。

### TC-ADMIN-006 系统设置

1. 点击左侧“系统设置”。
2. 预期显示考勤规则、项目参数、通知设置、数据维护等配置区域。
3. 修改规则或项目参数后点击保存。
4. 预期出现“已保存”类提示。
5. 点击导出、同步、清除类按钮。
6. 预期出现“功能开发中”提示或保持页面稳定。

## 9. 回归检查清单

- 页面无白屏、无未捕获错误。
- 登录态持久化与退出登录正常。
- 所有角色主要导航入口可达。
- Mock 数据加载成功，列表和统计不为空。
- 表单按钮禁用/启用逻辑合理。
- 弹窗可打开、关闭、提交。
- 移动端底部导航不遮挡主要操作。
- 后台侧栏高亮与面包屑随路由变化。

## 10. 已知风险

- 工人打卡目前是前端 `setTimeout` 模拟，没有调用真实打卡 API。
- Mock 模式下非 GET 请求统一返回成功，不代表真实后端写入成功。
- 报销和日报图片使用浏览器本地 blob URL，真实后端无法直接访问该 URL。
- `.env` 默认 `VITE_MOCK_ENABLED=false`，无后端时本地测试必须显式覆盖为 `true`。
